# insight

오늘 대화를 분석해서 `insights_{YYYYMMDD}.md` 파일을 프로젝트 루트에 생성한다.
날짜는 현재 날짜 기준으로 작성한다.

파일 구조는 아래 네 섹션으로 구성한다:

## 헤맸던 것 (삽질 포인트)
대화 중 막혔거나 잘못 이해했다가 교정된 개념, 예상과 다르게 동작했던 것들.
단순 나열이 아니라 "왜 헤맸는지", "어떻게 해결됐는지"까지 기록한다.
백엔드 경험이 위주인 개발자가 프론트/Next.js 생태계에서 겪는 낯선 점 위주로 기록한다.
관련 코드가 있으면 `파일경로:라인번호` 형식으로 참조를 함께 적는다. (예: `src/components/CodeBlock.tsx:22`)

## 공부하면 좋을 것
오늘 다룬 주제와 연결되는 개념 중 더 깊이 알면 도움이 될 것들.
표면적으로 해결됐더라도 원리를 모르고 넘어간 부분, 다음에 비슷한 문제를 만났을 때 스스로 해결할 수 있게 해주는 배경지식 위주로 작성한다.
특정 파일/코드와 연결되는 개념이면 `파일경로:라인번호` 참조를 함께 적는다.

## 알아두면 좋은 용어
대화 중 나온 프론트엔드/Next.js 생태계 특유의 용어나 개념, 그리고 UX/UI 용어.
- 프론트엔드/Next.js 용어는 백엔드 개발자 관점에서 대응하는 개념과 함께 설명한다. (예: "ISR ≈ 백그라운드 캐시 갱신")
- UX/UI 용어는 대화 중 말로 설명했지만 정확한 명칭을 몰랐던 것들을 정리한다. (예: "양 끝이 반원인 긴 막대 = Pill / Capsule 형태, Tailwind: rounded-full")

## 예습 거리 (다음에 구현할 것들)
대화에서 언급됐거나 자연스럽게 이어질 다음 구현 주제들.
각 항목마다 어떻게 접근하면 되는지 간략한 방향도 함께 적는다.
