# review

현재 변경된 코드를 아래 기준으로 검토해줘.
문제가 없는 항목은 건너뛰고, 문제 있는 것만 **파일명 + 라인 + 이유 + 수정 방향** 으로 알려줘.

---

## 1. 타입 안전성
- `any` 타입 사용 여부
- 함수 인자 / 반환값 타입 누락
- Notion API 응답을 그대로 쓰고 있는지 (mapper 거쳤는지)

## 2. 레이어 규칙 준수
- Notion API 직접 호출이 `lib/notion/repository.ts` 밖에 있는지
- 컴포넌트에서 직접 fetch / API 호출 여부
- service를 거치지 않고 repository를 바로 호출하는지

## 3. 단일 책임
- 하나의 함수가 너무 많은 일을 하는지 (기준: 30줄 초과 시 검토)
- 컴포넌트가 UI 외 로직을 포함하는지

## 4. 기타
- 환경변수 하드코딩 여부
- `console.log` 등 디버그 코드 잔존 여부
- ISR `revalidate` 설정 누락 여부 (페이지 컴포넌트 한정)
